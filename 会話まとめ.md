## 7. LINE配信設計

### 全体導線

```
新規導線：Instagram・広告 → ハブLINE → 新規メインLINE → 予約 → 来院 → 既存患者LINE
チラシ導線：チラシQR → WEBフォーム or LINE予約 → 来院前確認 → 来院 → 既存患者LINE
```

### アカウント構成

| LINE | 役割 |
|------|------|
| ハブLINE | 入口・メインLINEへの誘導のみ |
| 新規メインLINE | 教育・予約獲得・来院前フォロー |
| 既存患者LINE（新） | 予約管理・再来院促進・価格帯別配信 |
| 既存患者LINE（旧） | チラシ予約専用に転用 |

### 新規メインLINEの分岐

- **今すぐ予約したい** → `新規_予約直行_4通`シナリオ
- **腰チェックを見たい** → `新規_腰チェック_7通`シナリオ

### 重要な運用ルール

1. 新規LINEと既存患者LINEは必ず分ける
2. 予約が入ったら教育シナリオを止める（`新規_予約済`タグ）
3. 来院後は必ず既存患者LINEへ移す
4. 既存患者LINEでは価格帯（6,000円・5,000円）を混ぜない
5. 前日リマインドはUTAGEシナリオではなく手動一斉配信で対応
