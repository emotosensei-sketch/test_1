# 運用ルール

## このリポジトリについて
- このリポジトリは整体院運営の**第二の脳**として育てていく
- 整体院に関する情報を蓄積していく場所として使う

## ブランチ運用の原則
- **mainが唯一の正解**。KPI・タスク・基本情報は常にmainで更新する
- 各ブランチはテーマ別の独自ファイルだけを持つ（mainのコピーは持たない）
- mainを更新しても各ブランチには自動反映しない。mainが常に最新
- 新しいテーマが増えたらブランチを追加する

## ブランチ構成
| ブランチ | 用途 | 主な担当 |
|---------|------|---------|
| main | 全体ハブ・KPIダッシュボード・タスク管理 | 全員 |
| branch-b | MEO戦略・外部ポータルサイト掲載 | 幸生 |
| branch-c | 新規患者管理・来院記録・分析 | 幸生 |
| branch-sns | YouTube・スレッズ・X・ブログ運用方針 | 幸生・妻 |

## Claudeへの指示のしかた
作業内容に応じて読むブランチを指定する：

| 作業内容 | 指示の例 |
|---------|---------|
| 集客・戦略・タスク全般 | 「第二の脳メイン.mdを読んで続きから始めて」 |
| 新規患者の記録・分析 | 「第二の脳メイン.mdとbranch-cの新規患者管理フォルダを読んで続きから始めて」 |
| MEO・外部掲載 | 「第二の脳メイン.mdとbranch-bを読んで続きから始めて」 |
| SNS運用方針の検討 | 「第二の脳メイン.mdとbranch-snsの07_SNS運用フォルダを読んで続きから始めて」 |

## 更新ルール
- 新しい戦略・気づきが生まれたら該当ファイルに追記する
- KPIが変わったら第二の脳メイン.mdのダッシュボードを更新する
- タスクが完了したら「未」→「✅」に変更する
- 担当者が作業したら担当列の内容を確認・更新する

## 夫婦の役割分担
- タスク表の「担当」列で管理する
- 担当が変わる場合はタスク表を更新する
- ブランチで役割を分けない（情報が断絶するため）
- **Claudeを主に操作するのは妻**

| 担当 | 主な役割 |
|------|---------|
| 幸生 | 施術・売上管理・ポスティング・方針決定・患者とのやりとり |
| 妻 | SNS集客・SNSコンテンツ作成・資料作成・LINE構築・Claude操作 |

### 業務フロー
```
幸生が方針・要望を伝える
  ↓
妻がClaudeと一緒に形にする（コンテンツ・資料・戦略文書）
  ↓
幸生が確認・承認して実行
```
